# Tuesday 3/17 — Production Deploy

**Theme:** Get the backend live on Fly.io with all production secrets. By EOD you should have a real HTTPS API URL.

> ⚠️ Prerequisite: All Batch 5 Codex PRs must be merged before starting this day.

---

## Engineering Checklist

### 1. Install Fly CLI (if not already installed)
```bash
brew install flyctl
# or
curl -L https://fly.io/install.sh | sh
```

### 2. Authenticate with Fly.io
```bash
fly auth login
# Opens browser, sign in or create account at fly.io
```

### 3. Launch the app (first time only)
```bash
# From the repo root (where fly.toml lives)
fly launch --no-deploy
# When prompted:
#   App name: doordrill-api
#   Region: iad (US East — closest to Supabase us-east-1)
#   Don't add a Postgres or Redis database — we use Supabase + external Redis
```

### 4. Set all production secrets on Fly
Run these one at a time, substituting real values:
```bash
fly secrets set DATABASE_URL="postgresql://postgres.dlhmuoyeuttzrinmvrkt:[PASSWORD]@aws-1-us-east-1.pooler.supabase.com:5432/postgres?sslmode=require"
fly secrets set REDIS_URL="redis://:yourpassword@your-redis-host:6379/0"
fly secrets set JWT_SECRET="$(openssl rand -hex 32)"
fly secrets set OPENAI_API_KEY="sk-..."
fly secrets set DEEPGRAM_API_KEY="..."
fly secrets set ELEVENLABS_API_KEY="..."
fly secrets set ELEVENLABS_VOICE_ID="..."
fly secrets set SENTRY_DSN="https://abc123@o123.ingest.sentry.io/456"
fly secrets set AUTH_REQUIRED="true"
fly secrets set RESEND_API_KEY="re_..."
```

> ℹ️ `JWT_SECRET`: The command above generates a cryptographically random 32-byte hex string. Save it somewhere safe (1Password, etc.) — you'll need it if you ever redeploy from scratch.

> ℹ️ Need Redis? Easiest option at launch: [Upstash](https://upstash.com) — free tier, serverless Redis, copy the `redis://` connection string they give you.

### 5. Run alembic migration against production DB
```bash
cd backend
# Temporarily point at production DB to run migration
DATABASE_URL="postgresql://postgres.dlhmuoyeuttzrinmvrkt:[PASSWORD]@aws-1-us-east-1.pooler.supabase.com:5432/postgres?sslmode=require" \
  alembic upgrade head
# Should apply migration 0033 (password_reset_tokens)
```

### 6. Deploy
```bash
# From repo root
fly deploy
# First deploy takes 3–5 minutes (building Docker image)
# Watch the logs — look for "Starting worker processes"
```

### 7. Verify deploy
```bash
# Check status
fly status

# Hit the health check
curl https://doordrill-api.fly.dev/health
# Expected: {"status":"ok"}

# Check logs if anything's wrong
fly logs
```

### 8. Run E2E smoke test against production
```bash
cd backend
BASE_URL=https://doordrill-api.fly.dev python scripts/e2e_smoke_test.py
# Target: 22/24 passing (same as local)
```

### 9. Wire Resend into email provider
- [ ] Open `backend/app/core/email_provider.py` (or wherever email sending lives)
- [ ] Confirm it reads `RESEND_API_KEY` from settings
- [ ] If email provider isn't implemented yet, create a simple one:

```python
# backend/app/core/email_provider.py
import resend
from app.core.config import get_settings

async def send_email(to: str, subject: str, body: str) -> None:
    settings = get_settings()
    if not settings.resend_api_key:
        print(f"[EMAIL SKIP] To: {to} | Subject: {subject}")
        return
    resend.api_key = settings.resend_api_key
    resend.Emails.send({
        "from": "DoorDrill <noreply@yourdomain.com>",
        "to": to,
        "subject": subject,
        "text": body,
    })
```
