## Part 1 — Physical Device QA (Morning — ~2 hours)

Do this on a **real iPhone** and a **real Android device** (not simulator). Push notifications and audio don't work reliably in simulator.

### Fresh Install Test
- [ ] Delete the app from your phone if it's already installed
- [ ] Install via `eas build --profile preview` or Expo Go development build
- [ ] Go through the full onboarding flow as a new user

### Auth Flow
- [ ] Register a new account with a test email
- [ ] Log out, log back in
- [ ] Tap "Forgot password?" → enter email → confirm email arrives → tap link → reset password → log in with new password
- [ ] Let the JWT expire (or test silent refresh logic) — confirm you're not logged out unexpectedly

### Core Drill Flow
- [ ] Start a new drill session
- [ ] Speak at least 3 turns naturally
- [ ] End the session
- [ ] Wait for scorecard (up to 2 minutes) — confirm it loads with category scores
- [ ] Verify push notification arrives when score is ready

### Manager Flow (switch to a manager account)
- [ ] Log in as a manager
- [ ] Confirm reps appear in dashboard
- [ ] View a rep's completed session and score
- [ ] Send a manager invite to a new email address

### Edge Cases
- [ ] Put the app in the background mid-drill — does it handle reconnection gracefully?
- [ ] Turn on airplane mode during a drill — confirm it shows an error, not a crash
- [ ] Open the deep link `doordrill://reset-password?token=testtoken` — confirm it opens ForgotPasswordScreen in RESET state (token will be invalid but screen should open)

### Pass/Fail
- [ ] All flows work on iOS ✅ or note failures: `_________________________________`
- [ ] All flows work on Android ✅ or note failures: `_________________________________`

> If you find blockers: fix them before building. Don't submit a broken build — Apple reviewers test the app and will reject it.

## Part 2 — EAS Production Builds (After QA passes)

```bash
cd mobile

# Build both platforms simultaneously
eas build --platform all --profile production

# Or build separately:
eas build --platform ios --profile production
eas build --platform android --profile production
```

- First production build takes 15–30 minutes (EAS cloud build)
- You'll get an email when each build finishes
- Download links appear in https://expo.dev → your project → Builds

- [ ] iOS build finished — status: ✅ / ❌
- [ ] Android build finished — status: ✅ / ❌

### If a build fails
Common causes:
- Missing provisioning profile → run `eas credentials` again
- `google-services.json` not found → confirm it's at `mobile/android/app/google-services.json`
- PrivacyInfo.xcprivacy not found → confirm it's at `mobile/ios/PrivacyInfo.xcprivacy`
- Build error in app code → check EAS build logs for the specific error

## Part 4 — Submit iOS to App Store Review

```bash
# Option A — EAS Submit
eas submit --platform ios --profile production
# This uploads the .ipa to App Store Connect automatically

# Option B — Transporter app (Mac App Store, free)
# Download the .ipa from expo.dev → drag into Transporter → Submit
```

In App Store Connect after the build uploads (~10 min):
- [ ] Go to your app → iOS App → `+` next to Version
- [ ] Select the build that just uploaded
- [ ] Fill in **What's New** (first version — can say "Initial release")
- [ ] Answer **Export Compliance**: Does your app use encryption? → **No** (unless you added custom crypto — standard HTTPS doesn't count)
- [ ] Check all tabs are complete: App Information, Pricing, Review Information
- [ ] **Review Information** tab:
  - Demo account: `demo@doordrill.com` / `DoorDrill2026!` (or whatever test credentials you want to give Apple — create this account in production)
  - Notes: "DoorDrill is a sales training app. Create an account or use the demo credentials above to test. The app requires microphone access to run drill sessions."
- [ ] Hit **Submit for Review**

Expected timeline: 24–72 hours for first review decision.
