# Security Policy

## Supported Branch

Security fixes are applied against `main`. If a vulnerability affects an older feature branch, rebase or merge that branch onto the current `main` before validating the fix.

## Reporting a Vulnerability

Do not open public GitHub issues for security-sensitive reports.

Use GitHub's private vulnerability reporting flow for this repository if it is enabled. If private reporting is not available, contact the repository owner through an existing private channel and include:

- a clear description of the issue
- reproduction steps or a proof of concept
- impact assessment
- any relevant environment details

Please allow time for triage and coordinated remediation before disclosing details publicly.
